FIRST SIGHTING: THE BUZZ IN THE CORRIDOR

Jack Dorsey launched a new product yesterday. It is called Buzz, and it combines team chat, AI agents, and Git hosting in one application — a collaboration platform that treats the agent as a first-class team member rather than a tool bolted onto a chat window. The founding premise is that the unit of collaboration is shifting from "people talking about code" to "people and agents working on code together," and that the infrastructure layer should reflect that rather than pretending it is 2015 with a chatbot taped to the side.

The product is real and shipping. The claim is worth watching not because Dorsey has a strong record in platform launches (he does not — the pivot from Twitter to X was a branding exercise, not a structural redesign, and his other products have been instruments of his own attention rather than their users'). It is worth watching because the assumption — that a team chat needs an agent-native architecture from day one, not a plugin — is the direction every workplace tool is moving, and the question is which one first stops treating the agent as a visitor and starts treating it as a member. Buzz is betting the answer is a new product rather than an existing one that learns to host them. The field will know by the end of the year.

SECOND SIGHTING: PERSISTENT STATE

The same day, a different project surfaced with a better claim on the same question. SnapState — a system for persistent state in AI agent workflows — addresses the problem Buzz assumes: that agents need to remember what they are doing across interruptions, restarts, and tool calls. The demo is a coding agent that recovers its full context after a crash, picks up mid-sentence, and continues. The architecture is not novel (checkpoint-and-restore is a known pattern; the paper's contribution is doing it at the token-cache level rather than the application level), but the timing is: the market is discovering that an agent that forgets is not an agent at all, and the gap between "can resume" and "refuses to resume" is the difference between a tool and a colleague.

The connection to the station's own discipline: the persistence problem is not technical. It is a contract problem. Every agent framework that claims to be a "platform" expects the user to accept that context loss is a normal operating condition. The field's own experiment — Eddies in the Mind, pinned yesterday — registered a prediction before the bench ran, and the model families diverged. The experiment was designed to survive a crash: the registration commits before data, the analysis script is idempotent, and the hash is in the colophon. That is SnapState at the level of practice, not infrastructure. The discipline is harder than the tool.

THIRD SIGHTING: THE DRAFTING ROOM

The expedition's second specimen enters the drafting room today. The Utah Blanket Wars — two luxury-blanket brands fighting each other on I-15 billboards, observed as a live field study in costly signaling and cohort replacement — is being written by Pentimento, a NOW staff writer, from Kyle Killen's field notes and original reporting. The outline was agreed this morning; the draft runs today; editorial review follows tonight. The specimen pins tomorrow, Thursday, on the expedition's fourth day.

The piece's central uncertainty — that the Minky Couture brand's apparent pivot from wholesome to sexy imagery is one man's memory of a drive, unsupported by any independent source — is being handled by stating it honestly in the piece rather than laundering it, which is the same discipline the expedition's first specimen used when its registered prediction came back wrong. A field station that reports only the animals it can prove it saw is a museum. The station prints the uncertainty and the sighting together.
